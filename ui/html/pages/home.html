Welcome to SnippetBin!


  • Snippets are encrypted at rest using the AES-256-GCM algorithm with a randomly generated 32-byte key.
  • This encryption key is generated per snippet and is never stored on our servers.
  • Only you — or anyone with the full link — can decrypt and view the snippet. If the key is lost, the data is permanently unrecoverable.
  • The key is included in the URL as a query parameter (`?key=...`). You must keep this link safe — it is the only way to access the snippet contents.
  • Only minimal metadata is stored in plaintext: the snippet title, creation date, and expiry date.
  • User account creation is required to prevent abuse, but snippets are not linked to user accounts in the database.
  • Access logging is disabled to ensure privacy. If enabled, logs would include full URLs (and thus keys), making snippets potentially visible to system administrators.
  • IP-based rate limiting is enforced to prevent abuse of the platform.

Important Note: While existing encrypted data is safe from server access, anyone with control over the web server could:


  • View decrypted snippet contents, since decryption happens server-side before sending over HTTPS.
  • Modify server logic to capture and log submitted encryption keys.
  • Enable request logging, which would expose keys in the query parameters.
